プロンプト設計 — 理解確認クイズ

Lesson の内容を確認しましょう。全10問

📝 全 10 問 「答えを見る」ボタンで各問の正解を確認できます
Q1
「再現性のあるプロンプト」を作るために最も重要な考え方はどれですか？
✅ 正解: タスク・制約・フォーマットを明示的に構造化する
再現性のあるプロンプトには「何をするか（タスク）」「どのような制約か」「どの形式で出力するか」の3点が必要です。temperature=0は決定論的にはなりますが、プロンプト自体が曖昧であれば品質は保証されません。
Q2
Few-shot プロンプトとは何ですか？
✅ 正解: 入力と期待出力のペアを例としてプロンプトに含める手法
Few-shot プロンプトは「このような入力にはこのような出力を返してほしい」という例（shot）をプロンプト内に複数含めることで、LLMがタスクの意図を正確に理解できるようにする手法です。
Q3
OpenAI API で response_format={"type": "json_object"} を指定する目的はどれですか？
✅ 正解: LLM の出力を必ず JSON 形式にすることで後続処理を安定させる
response_format={"type": "json_object"} を指定すると、LLM は必ず有効な JSON を返します。これにより json.loads() でのパースが安全になり、後続の処理コードが安定します。
Q4
Pydantic モデルを使った Structured Outputs の最大のメリットはどれですか？
✅ 正解: スキーマ定義によりフィールド欠損や型違いが原理的に発生しない
Pydantic モデルで定義したスキーマに対して Structured Outputs を使うと、LLM の出力がそのスキーマに完全準拠することが保証されます。フィールドの欠損や型の不一致が発生しないため、後続処理が型安全になります。
Q5
プロンプトに「禁止事項」を書く主な目的はどれですか？
✅ 正解: LLM が余計な情報を付加したり範囲外の回答をするのを防ぐ
LLM は「役に立とう」とするあまり、求めていない情報を付け加えたり、許容範囲を超えた回答をすることがあります。禁止事項を明示することで、意図しない誤動作を防ぐことができます。
Q6
Anthropic（Claude）が推奨するプロンプトの4分割スタイルはどれですか？
✅ 正解: 指示・コンテキスト・例・出力形式
Anthropic の推奨スタイルは「指示（Task）・コンテキスト（Context）・例（Examples）・出力形式（Output Format）」の4分割です。この構造に従うことで、Claude が意図を正確に理解し高品質な出力を生成しやすくなります。
Q7
プロンプトに「評価基準」を含める効果として正しいものはどれですか？
✅ 正解: LLM が自己チェックしながら出力を改善できる
「良い出力の条件」をプロンプト内に記述することで、LLM は生成中にそれらの条件を満たしているかを考慮しながら出力を生成します。これにより品質が向上しますが、完全な正確性を保証するものではありません。
Q8
プロンプト設計の「明確・具体的・検証可能」という3原則のうち、「検証可能」とは何を意味しますか？
✅ 正解: 出力が要件を満たしているか自動テストできる形にすること
「検証可能」とは、出力がプロンプトで定めた要件（フォーマット・内容・制約）を満たしているかをコードで自動検証できる状態を指します。JSON形式の出力はスキーマ検証が容易で、検証可能性が高いです。
Q9
出力フォーマットとして JSON を指定する際、システムプロンプトに必要な記述はどれですか？
✅ 正解: 「JSONで回答してください」とフィールド定義を含める
response_format={"type": "json_object"} を指定する場合、システムプロンプトに「JSON で回答してください」という指示が必要です。さらにフィールド名・型・説明を含めることで、期待するスキーマの JSON が得られます。
Q10
「役割付与（ペルソナ指定）」と「タスク定義」を比較したとき、実務上より効果的なのはどちらですか？
✅ 正解: 具体的なタスク定義の方が出力品質への影響が大きい
「あなたは〇〇のエキスパートです」という役割付与より、「以下の〇〇を行い、△△の形式で出力してください」という具体的なタスク定義の方が出力品質に大きく影響します。役割付与は補助的に使うことができますが、それだけに頼るのは避けましょう。
