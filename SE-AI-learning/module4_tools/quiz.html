M4 QUIZ

ツール利用（業務システムと繋ぐ）— 理解確認クイズ

Lesson の内容を確認しましょう。全10問

📝 全 10 問 「答えを見る」ボタンで各問の正解を確認できます
Q1
Function Calling（ツール利用）においてツール定義に必要な3つの要素はどれですか？
✅ 正解: name・description・parameters
ツール定義は name（関数名）・description（LLMが使用判断に使う説明文）・parameters（引数のJSONスキーマ定義）の3要素で構成されます。特に description の品質がLLMの使用判断精度に直接影響します。
Q2
Function Calling の3ステップとして正しいものはどれですか？
✅ 正解: ①LLMがtool_callsを返す → ②アプリが関数実行 → ③結果をLLMに渡して最終回答
Function Callingでは、LLMは実際にAPIを呼び出しません。LLMは「どの関数をどの引数で呼ぶか」を tool_calls として返すだけです。実際の関数実行はアプリケーション側（コード）が行い、その結果を role: "tool" として返すと最終回答が生成されます。
Q3
ツール定義の description が重要な理由はどれですか？
✅ 正解: LLMがツールをいつ・どのように使うかの判断基準になるから
LLMはツールの description を読んで「このツールを使うべきか」「どの引数を渡すか」を判断します。説明が曖昧だと誤った使用につながります。使用条件・引数の意味・返り値の説明を具体的に書くことが重要です。
Q4
「段階的権限付与」において、最初に許可すべき操作はどれですか？
✅ 正解: 読み取り（search, get, list）から始める
最小権限の原則に従い、まず読み取り専用の操作から始めます。誤動作してもデータが変更されないため安全です。書き込み（create）、更新・削除（update/delete）、外部送信（email）の順に慎重に権限を追加していきます。
Q5
Human-in-the-loop を実装すべき操作として最も適切なものはどれですか？
✅ 正解: 大量データの削除や顧客への一斉メール送信
Human-in-the-loop は「取り消しが困難な重要なアクション」に適用します。大量削除や顧客への送信は一度実行すると元に戻せないため、必ず人間の確認ステップが必要です。読み取りのみの操作にはHuman-in-the-loopは通常不要です。
Q6
MCP（Model Context Protocol）の主なメリットはどれですか？
✅ 正解: ツールをサーバーとして分離することで、再利用性・セキュリティ・保守性が向上する
MCPではツールをサーバーとして独立させるため、①複数のAIアプリから同一のMCPサーバーを利用できる（再利用性）、②サーバー側で認証・権限管理を一元化できる（セキュリティ）、③ツールの更新がサーバー側だけで完結する（保守性）というメリットがあります。
Q7
FastMCP でツールを定義する際に使う Python の構文はどれですか？
✅ 正解: @mcp.tool() デコレータを関数に付ける
FastMCP では @mcp.tool() デコレータを Python 関数に付けるだけでツールとして公開できます。関数の型アノテーションと docstring が自動的にツールの parameters と description に変換されます。
Q8
OWASP LLM06「過剰な権限（Excessive Agency）」とはどのようなリスクですか？
✅ 正解: LLM エージェントに過剰な権限を与えることで、誤動作・悪意ある入力による被害が大きくなるリスク
OWASP LLM06では、エージェントに必要以上の権限（削除権限・外部送信権限など）を与えることで、プロンプトインジェクションや誤動作の際に取り返しのつかない被害が生じるリスクを指摘しています。最小権限の原則で対策します。
Q9
監査ログに記録すべき情報として最も適切なものはどれですか？
✅ 正解: タイムスタンプ・ユーザーID・呼び出したツール名・引数・実行結果
監査ログには「いつ（タイムスタンプ）・誰が（ユーザーID）・何を（ツール名）・どのような設定で（引数）・どうなったか（実行結果ステータス）」を記録します。パスワードやAPIキーは機密情報のためログに含めてはいけません。
Q10
「エージェントに与えるツール数は最小限にする」原則の主な理由はどれですか？
✅ 正解: ツール数が増えると LLM が誤ったツールを選択するリスクが増え、セキュリティリスクも拡大するから
ツール数が増えると①LLMが文脈に合わないツールを誤って選ぶリスクが増加し、②攻撃者がプロンプトインジェクションで悪用できるツールの選択肢も増えます。最小権限の原則に従い、タスクに必要なツールのみを提供することが安全設計の基本です。
