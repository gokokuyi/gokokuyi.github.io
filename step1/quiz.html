Step 1 確認テスト — Claude Code 入門

所要時間: 約10分  |  問題数: 10問

選択問題

Q1. Claude Code とは何ですか？

正解: b — Anthropic が提供する公式 AI CLI（コマンドラインインターフェース）ツールです。

Q2. Claude Code をインストールするコマンドはどれですか？

正解: c — npm でグローバルインストールします。Node.js 18 以上が必要です。

Q3. Claude Code を起動するコマンドはどれですか？

正解: c — プロジェクトディレクトリで claude と入力して起動します。

Q4. Claude Code が利用できる環境として誤っているものはどれですか？

正解: c — Microsoft Word アドインは存在しません。CLI、デスクトップアプリ、Web アプリ、VS Code 拡張、JetBrains 拡張で利用できます。

Q5. プロジェクト固有の設定を Claude Code に伝えるためのファイルはどれですか？

正解: b — プロジェクトルートに置く CLAUDE.md で、コーディング規約・禁止事項などをプロジェクト固有に設定できます。

Q6. Claude Code の会話中に改行を入力するショートカットはどれですか？

正解: c — Shift + Enter で改行、Enter 単独でメッセージ送信です。

Q7. CLAUDE.md ファイルの読み込み優先順位として正しいものはどれですか？

正解: c — より具体的な場所にある CLAUDE.md が優先されます。サブディレクトリ（最高）→ プロジェクトルート → グローバル（最低）の順です。

Q8. Claude Code で特に確認が重要な操作はどれですか？

正解: c — 外部への影響がある操作（git push、PR 作成、ファイル削除等）は元に戻しにくいため、特に確認が重要です。

Q9. セッションをリセットするスラッシュコマンドはどれですか？

正解: c — /clear で会話履歴をリセットします。コンテキストが長くなりすぎたときに使います。

Q10. ベストプラクティスでないものはどれですか？

正解: b — 本番 API キーを CLAUDE.md に記載してはいけません。CLAUDE.md は git リポジトリに含まれることが多く、機密情報の漏洩につながります。
