1 Claude Code とは

Claude Code は Anthropic が提供する公式 AI CLI（コマンドラインインターフェース）ツールです。AI アシスタント Claude をターミナルや IDE から直接利用でき、ソフトウェアエンジニアリング作業を強力にサポートします。

主な特徴

機能説明
コード生成・修正自然言語でコードを生成、バグを修正
ファイル操作コードベースを読み込み、分析・編集
コマンド実行テスト実行、ビルド、git 操作などを自動化
コードベース理解プロジェクト全体の構造を把握して回答
Web 検索最新情報を調べながら回答

利用できる環境

環境起動方法
CLI（ターミナル）claude コマンドで起動
デスクトップアプリMac/Windows 向けアプリ
Web アプリclaude.ai/code
VS Code 拡張IDE 内で利用
JetBrains 拡張IDE 内で利用

従来の AI チャットとの違い

従来の AI チャット（ChatGPT 等）:
  - ブラウザで会話
  - コードを貼り付けて質問
  - 修正案をコピーして手動で適用

Claude Code:
  - ターミナル・IDE で動作
  - プロジェクト全体を自動で読み込み
  - ファイルを直接編集・コマンドを実行

2 インストールと初期設定

前提条件

  • Node.js 18 以上がインストールされていること
  • Anthropic アカウント（claude.ai）を持っていること
  • 基本的なターミナル操作ができること

インストール手順

# npm でグローバルインストール
npm install -g @anthropic-ai/claude-code

# インストール確認
claude --version

初回起動と認証

# 起動（初回はブラウザで認証画面が開く）
claude

初回起動時の手順：

  1. ターミナルで claude を実行
  2. ブラウザが自動で開き、Anthropic アカウントでログイン
  3. 認証完了後、ターミナルに戻って会話開始

API キーを使う場合

# 環境変数で設定
export ANTHROPIC_API_KEY="sk-ant-xxxxxxxxxxxx"

# または .bashrc / .zshrc に追記して永続化
echo 'export ANTHROPIC_API_KEY="sk-ant-xxxx"' >> ~/.zshrc

4 はじめての会話

プロジェクトディレクトリで起動

cd my-project
claude

典型的な会話の流れ

You: このプロジェクトの全体的な構造を教えて

Claude: [ファイル構造を分析して説明します]
  - src/ にメインのロジック
  - tests/ にテストコード ...

You: src/utils.py の中に潜在的なバグがあると思う。確認して

Claude: 35行目に None チェックが欠けています。修正しますか？

You: 修正して、テストも追加して

Claude: 修正完了しました。テストを実行して確認します...

効果的な指示の仕方

NGOK
「このコードを直して」「src/auth.py の login 関数が None を返すことがある。エラーハンドリングを追加して」
「関数を追加して」「メールアドレスを検証する関数を validators.py に追加して。RFC 5322 準拠で」

5 権限モードと安全性

確認が求められる操作の例

操作の種類説明
ファイル削除元に戻せない操作
git pushリモートへの影響
PR 作成・コメント他者への影響
パッケージのアンインストール環境変更
データベース操作データへの影響

安全な使い方のポイント

  1. 確認メッセージをちゃんと読む — 何を実行しようとしているか必ず確認
  2. git で管理されたプロジェクトで使う — 変更をいつでも元に戻せる
  3. 本番環境では慎重に — テスト環境で先に確認する
  4. CLAUDE.md で制約を設ける — 禁止操作を明示する

6 CLAUDE.md ファイル（プロジェクト設定）

プロジェクトルートに CLAUDE.md ファイルを置くことで、Claude Code の動作をプロジェクト固有にカスタマイズできます。

# CLAUDE.md の例

## 概要
Python で書かれた REST API サーバー（FastAPI）

## コーディング規約
- PEP 8 に準拠すること
- 型ヒントを必ず付けること

## テスト
- テストフレームワーク: pytest
- `pytest tests/` で実行

## 禁止事項
- main ブランチへの直接コミット禁止
- .env ファイルのコミット禁止

CLAUDE.md の読み込み順（優先度）

場所優先度
サブディレクトリ内の CLAUDE.md高（最優先）
プロジェクトルートの CLAUDE.md中
~/.claude/CLAUDE.md（グローバル）低
