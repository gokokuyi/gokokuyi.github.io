Step 4 チャレンジ — MCP クライアントを実装する

CHALLENGE ローカル実行 所要時間: 約30〜45分

このチャレンジのゴール
MCP（Model Context Protocol）サーバーに接続し、ツール定義を動的に取得・実行する クライアントコードを 3箇所 埋めて完成させる。

MCP（Model Context Protocol）とは

Step 4 で学んだ「Claude Code のツール拡張」の基盤となるプロトコルです。 従来の手書きツールと MCP の最大の違いは 「ツールの実装場所の分離」 にあります。

手書きツール（従来方式）MCP（動的取得）
ツール定義クライアントコードに手書きサーバーから動的取得
ツール実行自分の Python 関数を直接呼ぶMCP サーバーに委譲
ツールを追加したときクライアントコードも修正が必要クライアントは変更不要
サーバーの言語—Python / TypeScript / 任意
あなたが実装する部分
MCP クライアント ──── SSE ────> MCP サーバー
    ① ツール一覧を問い合わせる    <──────────  ツール定義を返す
    ② LLM にツール定義を渡す
    ③ ツール実行を委譲する        ──────────>  実行して結果を返す

Step 1: MCP サーバーを確認する（変更不要）

サーバーが提供する 3つのツール を把握しておきましょう。

ツール名説明主なパラメータ
get_current_datetime現在の日付・時刻を返すなし
calculate_loan_monthly_payment元利均等返済の月額を計算principal / annual_rate_percent / months
count_days_until今日から指定日までの残日数を計算target_date_str（YYYY-MM-DD）
📄 step4_mcp-server.py（クリックで展開） 
from datetime import date, datetime
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("SxxxFinanceTools")


@mcp.tool()
def get_current_datetime() -> str:
    """現在の日付と時刻を返す。"""
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")


@mcp.tool()
def calculate_loan_monthly_payment(
    principal: float, annual_rate_percent: float, months: int
) -> str:
    """元利均等返済方式での毎月の返済額を計算する。"""
    if annual_rate_percent == 0:
        monthly_payment = principal / months
    else:
        r = annual_rate_percent / 100 / 12
        monthly_payment = principal * r * (1 + r) ** months / ((1 + r) ** months - 1)
    total_payment = monthly_payment * months
    total_interest = total_payment - principal
    return (
        f"借入元金: {principal:,.0f} 円\n"
        f"年利: {annual_rate_percent} %\n"
        f"返済期間: {months} ヶ月（{months // 12} 年）\n"
        f"毎月の返済額: {monthly_payment:,.0f} 円\n"
        f"返済総額: {total_payment:,.0f} 円\n"
        f"支払利息合計: {total_interest:,.0f} 円"
    )


@mcp.tool()
def count_days_until(target_date_str: str) -> str:
    """今日から指定した日付までの残り日数を計算する。"""
    try:
        target = date.fromisoformat(target_date_str)
    except ValueError:
        return f"日付の形式が正しくありません。'YYYY-MM-DD' 形式で指定してください。"
    today = date.today()
    delta = (target - today).days
    if delta > 0:
        return f"今日（{today}）から {target_date_str} まで、あと {delta} 日です。"
    elif delta == 0:
        return f"今日（{today}）が {target_date_str} です。"
    else:
        return f"{target_date_str} は {abs(delta)} 日前（{today} 時点）です。"


if __name__ == "__main__":
    print("MCP サーバーを起動します（http://localhost:8000/sse）")
    mcp.run(transport="sse")

ヒント: session.list_tools() は非同期メソッドです。await を付けて呼び出してください。 戻り値の .tools 属性にツールオブジェクトのリストが入っています。

②

MCP ツール定義を OpenAI 形式に変換

難易度: 中

取得した MCP ツール定義を、OpenAI の Responses API が受け付ける形式（辞書のリスト）に変換してください。

step4_mcp-client.exercise.py — discover_tools() 続き EDIT HERE ② 
    # EDIT HERE ②: MCP のツール定義を OpenAI のツール定義形式に変換してください
    # MCP ツールのプロパティ   : tool.name / tool.description / tool.inputSchema
    # OpenAI ツールのプロパティ: type / name / description / strict / parameters
    openai_tools = ...  # ← ここを完成させる
    return openai_tools
ヒント — OpenAI ツール定義の形式: 
{
    "type": "function",
    "name": tool.name,           # MCP の tool.name をそのまま使う
    "description": tool.description or "",
    "strict": False,
    "parameters": tool.inputSchema,  # MCP の inputSchema が JSON Schema 形式
}
各ツールについてこの辞書を作り、リストに追加して返してください。
正解: 
openai_tools = []
for tool in tools_response.tools:
    logger.info(f"  - {tool.name}: {tool.description}")
    openai_tools.append({
        "type": "function",
        "name": tool.name,
        "description": tool.description or "",
        "strict": False,
        "parameters": tool.inputSchema,
    })
return openai_tools

サーバー側でツールを追加・変更しても、このクライアントコードは一切変更不要になります。これが MCP の最大のメリットです。

    #   # result.content はテキストブロックのリスト
    #   return "\n".join(block.text for block in result.content if hasattr(block, "text"))
    result = ...  # ← ここを完成させる
    return ...    # ← ここを完成させる
ヒント:
  • session.call_tool(ツール名, 引数辞書) でサーバーにツール実行を委譲します
  • 戻り値の result.content はテキストブロックのリストです

完成版コード（全体）

📄 step4_mcp-client.sample.py — 完成サンプルを見る（MockLLM 版） 
import asyncio
import json
import uuid
from dataclasses import dataclass, field
from typing import Any
import logging

from mcp import ClientSession
from mcp.client.sse import sse_client

logging.basicConfig(level=logging.INFO, format="%(message)s")
logger = logging.getLogger(__name__)

MCP_SERVER_URL = "http://localhost:8000/sse"


# ── モック LLM ──────────────────────────────────────────────
@dataclass
class MockFunctionCall:
    name: str
    arguments: str
    type: str = "function_call"
    call_id: str = field(default_factory=lambda: f"mock_{uuid.uuid4().hex[:8]}")

@dataclass
class MockResponse:
    output: list
    output_text: str = ""

class MockLLMResponses:
    _ROUTING = [
        (("ローン","返済","月々","月額","借入"),
         "calculate_loan_monthly_payment",
         {"principal": 3_000_000, "annual_rate_percent": 1.5, "months": 120}),
        (("今日","今","日付","日時","時刻","現在"),
         "get_current_datetime", {}),
        (("残り","あと何日","日数","期限","まであと"),
         "count_days_until", {"target_date_str": "2025-12-31"}),
    ]

    def create(self, *, input, tools, **_):
        tool_outputs = [m["output"] for m in input
                        if isinstance(m, dict) and m.get("type") == "function_call_output"]
        if tool_outputs:
            summary = "【モック回答】ツール実行結果:\n" + "\n---\n".join(tool_outputs)
            return MockResponse(output=[], output_text=summary)
        user_msg = next((m.get("content","") for m in reversed(input)
                         if isinstance(m,dict) and m.get("role")=="user"), "")
        available = {t["name"] for t in tools}
        for keywords, tool_name, args in self._ROUTING:
            if tool_name in available and any(kw in user_msg for kw in keywords):
                return MockResponse(
                    output=[MockFunctionCall(name=tool_name,
                                            arguments=json.dumps(args, ensure_ascii=False))],
                    output_text="")
        return MockResponse(output=[],
                            output_text=f"[MockLLM] 「{user_msg}」に対応するツールが見つかりませんでした。")

class MockLLMClient:
    def __init__(self): self.responses = MockLLMResponses()


# ── MCP ユーティリティ ──────────────────────────────────────
async def discover_tools(session: ClientSession) -> list[dict[str, Any]]:
    # ① ツール一覧を取得
    tools_response = await session.list_tools()
    logger.info(f"MCP サーバーから {len(tools_response.tools)} 件のツールを取得しました")
    # ② OpenAI 形式に変換
    openai_tools = []
    for tool in tools_response.tools:
        logger.info(f"  - {tool.name}: {tool.description}")
        openai_tools.append({
            "type": "function",
            "name": tool.name,
            "description": tool.description or "",
            "strict": False,
            "parameters": tool.inputSchema,
        })
    return openai_tools

async def execute_tool(session: ClientSession, output) -> str:
    args = json.loads(output.arguments)
    logger.info(f"MCP ツール実行: {output.name}({args})")
    # ③ ツール実行を委譲
    result = await session.call_tool(output.name, args)
    return "\n".join(block.text for block in result.content if hasattr(block, "text"))


# ── デモ実行 ─────────────────────────────────────────────────
async def run_demo(client: MockLLMClient, session: ClientSession) -> None:
    tools = await discover_tools(session)
    demo_questions = [
        "今日の日付と現在時刻を教えてください",
        "借入300万円・年利1.5%・10年ローンの月々の返済額を計算してください",
        "2025年12月31日まであと何日残っていますか",
    ]
    print("\n" + "=" * 60 + "\nMCP クライアント デモ（モック LLM 使用）\n" + "=" * 60)
    for question in demo_questions:
        print(f"\nUser: {question}")
        messages: list[Any] = [{"role": "user", "content": question}]
        for _ in range(3):
            response = client.responses.create(input=messages, tools=tools)
            messages += response.output
            tool_called = False
            for output in response.output:
                if output.type != "function_call": continue
                tool_called = True
                tool_result = await execute_tool(session, output)
                messages.append({"type": "function_call_output",
                                  "call_id": output.call_id, "output": tool_result})
            if not tool_called: break
        if response.output_text:
            print(f"Assistant: {response.output_text}")
    print("\n" + "=" * 60 + "\nデモ完了\n" + "=" * 60)

async def main() -> None:
    client = MockLLMClient()
    logger.info(f"MCP サーバーに接続中... ({MCP_SERVER_URL})")
    async with sse_client(url=MCP_SERVER_URL) as (read, write):
        async with ClientSession(read, write) as session:
            await session.initialize()
            logger.info("MCP セッション確立")
            await run_demo(client, session)

if __name__ == "__main__":
    asyncio.run(main())

学習のポイントまとめ

ポイント内容
ツールの動的取得session.list_tools() で実行時にサーバーから取得。クライアントの変更不要でツールを追加できる
ツール実行の委譲session.call_tool(name, args) でサーバー側の関数を呼び出す。言語・ホストを問わない
asyncio.run()MCP SDK は非同期設計。asyncio.run(main()) でエントリーポイントを起動する
Claude Code との関係Claude Code の MCP サーバー設定（settings.json）も同じ仕組みで外部ツールを Claude に提供している
Claude Code での活用:
~/.claude/settings.json の mcpServers に追加すると、自作 MCP サーバーのツールを Claude Code から直接利用できます。 
{
  "mcpServers": {
    "finance": {
      "command": "python",
      "args": ["step4_mcp-server.py"]
    }
  }
}
