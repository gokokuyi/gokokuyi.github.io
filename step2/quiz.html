Step 2 確認テスト — ファイル操作と基本ツール

所要時間: 約10分  |  問題数: 10問

選択問題

Q1. ファイルの内容を読み込むために使うツールはどれですか？

正解: b — Read ツールでファイルの内容を読み込みます。cat や head などの Bash コマンドより Read ツールを優先して使います。

Q2. ファイルを部分的に変更する際に推奨されるツールはどれですか？

正解: c — Edit ツールは特定箇所のみを差分として変更します。ファイル全体を送信しないため効率的です。

Q3. Write ツールと Edit ツールの違いとして正しいものはどれですか？

正解: b — Write はファイル全体を上書き（全コンテンツを渡す必要あり）、Edit は old_string → new_string の差分のみで変更します。

Q4. Glob パターン **/*.test.ts が検索するファイルはどれですか？

正解: b — ** はすべてのディレクトリ階層を意味します。どの深さにある .test.ts ファイルもマッチします。

Q5. Grep ツールで「マッチしたファイルのパスのみ」表示するモードはどれですか？

正解: b — output_mode: "files_with_matches" でマッチしたファイルパスのみを表示します（デフォルトモードでもあります）。

Q6. Bash ツールを使わずに専用ツールを使うべき理由として誤っているものはどれですか？

正解: c — 速度が優れているとは限りません。専用ツールを使う主な理由は「ユーザーがレビューしやすい」「意図が明確になる」「エラーが起きにくい」点です。

Q7. ファイルを編集する前に Read ツールで確認することが重要な理由はどれですか？

正解: b — 既存コードを把握せずに編集すると、意図しない変更や既存ロジックの破壊につながります。Read で内容を確認してから編集することが重要です。

Q8. src/**/*.{js,ts} という Glob パターンはどのファイルにマッチしますか？

正解: b — src/** で src 以下の全階層、{js,ts} で .js または .ts の両方にマッチします。

Q9. Bash ツールで「対話型コマンド」が使えない主な理由はどれですか？

正解: b — Claude Code は自動化環境で動作するため、git rebase -i のような対話型入力を待つコマンドは使用できません。

Q10. 一部だけ変更したいときに Write ではなく Edit を使うべき理由はどれですか？

正解: c — Write を使うと Claude がファイル全体を再生成するため、他の箇所を誤って変更したり削除するリスクがあります。部分変更には Edit を使います。
