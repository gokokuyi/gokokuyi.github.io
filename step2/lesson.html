1 Claude Code のツール概念

Claude Code は「ツール」を通じてファイル操作やシステム操作を行います。ユーザーが自然言語で依頼すると、Claude が適切なツールを選択して実行します。

2 Read — ファイルを読む

ファイルの内容を読み込むツールです。Claude Code が作業を始める前に必ず使います。

You: config.yaml を読んで設定内容を説明して

Claude: 設定ファイルには以下が含まれています：
  - データベース: PostgreSQL（ポート 5432）
  - キャッシュ: Redis（ポート 6379）

Read ツールの特徴

  • 行番号付きで表示されるため、正確な位置を参照できる
  • 大きなファイルは自動的に分割して読み込む
  • 画像ファイル（PNG, JPG 等）も読み込める
  • PDF ファイルも対応（ページ指定可能）
  • Jupyter Notebook（.ipynb）も読み込める

3 Write — ファイルを作成・上書き

新しいファイルを作成するか、既存ファイルを完全に上書きします。

You: tests/test_utils.py という新しいテストファイルを作成して

Write vs Edit の使い分け

状況使うツール
新しいファイルを作るWrite
ファイルを丸ごと書き直すWrite
ファイルの一部だけ変更するEdit（推奨）
注意: Write は既存のファイルを完全に上書きします。一部だけ変更したい場合は Edit を使うのが安全です。

4 Edit — ファイルを部分編集

既存ファイルの特定箇所だけを変更するツールです。変更差分のみを送るため、大きなファイルでも効率的です。

You: src/auth.py の login 関数に None チェックを追加して

# 変更前
def login(username, password):
    user = db.find_user(username)
    return authenticate(user, password)

# 変更後（Edit で追加）
def login(username, password):
    if not username:
        raise ValueError("username is required")
    if not password:
        raise ValueError("password is required")
    user = db.find_user(username)
    return authenticate(user, password)

5 Glob — ファイルを検索

ファイル名のパターンでファイルを検索するツールです。

パターン記法

パターン意味例
*任意の文字列（同一ディレクトリ）*.py
**任意のディレクトリ階層**/*.py
?任意の1文字test?.py
{a,b}a または b*.{js,ts}

よく使う指示例

指示パターン
全テストファイルを探す**/*test*.py
設定ファイルを探す**/*.{yaml,yml,json,toml}
src の TS ファイルを探すsrc/**/*.ts

6 Grep — ファイル内を検索

ファイルの内容を正規表現で検索するツールです。

You: "TODO" コメントがどこにあるか探して

Claude: 以下の箇所に TODO コメントが見つかりました：
  - src/utils.py:45 - TODO: エラーハンドリングを追加
  - src/auth.py:23  - TODO: ログを追加

Grep の検索モード

モード説明
files_with_matchesマッチしたファイルパスのみ表示（デフォルト）
contentマッチした行の内容を表示
countマッチ数を表示

7 Bash — コマンドを実行

シェルコマンドを実行するツールです。テスト実行、パッケージインストール、git 操作などに使います。

You: テストを実行して結果を教えて

Claude: テスト結果：
  - 通過: 45件
  - 失敗: 2件
  失敗の詳細: test_auth.py::test_login - AssertionError: ...

Bash ツールの注意点

  • 破壊的なコマンド（rm -rf, drop table 等）は確認を求められる
  • 長時間のコマンドはタイムアウトがある（最大10分）
  • 対話型コマンド（vim, less 等）は使えない

8 ツールの使い分けガイド

やりたいこと使うツール
ファイル名で探すGlob
内容で探すGrep
ファイルを読むRead
一部だけ変更するEdit（推奨）
全部書き直し / 新規作成Write
コマンドを実行するBash

専用ツールを優先すべき理由

Bash でやる場合専用ツールを使う場合
cat file.pyRead ツール
find . -name "*.py"Glob ツール
grep -r "TODO" .Grep ツール
sed -i 's/foo/bar/' file.pyEdit ツール

専用ツールの方がユーザーがレビューしやすく、エラーが起きにくく、意図が明確になります。
