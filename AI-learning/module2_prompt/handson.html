Step 1

基本的な構造化プロンプト ─ JSON 形式で回答させる

顧客の問い合わせ文を受け取り、カテゴリ・緊急度・要約を JSON 形式で返すプロンプトを作成してください。

# Step 1: JSON 形式の構造化プロンプト
inquiry = "システムにログインできず、業務が止まっています。今すぐ対応してください。"

system_prompt = """  # EDIT HERE!
以下の顧客問い合わせを分析し、必ず JSON 形式で回答してください。

## 出力形式
{
  "category": "技術" | "契約" | "一般",
  "urgency": "高" | "中" | "低",
  "summary": "50字以内の要約",
  "suggested_action": "推奨対応"
}
"""

response = client.chat.completions.create(
    model=MODEL,
    response_format={"type": "json_object"},
    messages=[
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": inquiry}
    ]
)

result = json.loads(response.choices[0].message.content)  # EDIT HERE!
print(f"カテゴリ: {result['category']}")
print(f"緊急度: {result['urgency']}")
print(f"要約: {result['summary']}")
ヒント ①
system_prompt には「JSON で回答してください」という指示と、出力フィールドの定義（名前・型・説明）を含めます
フィールド定義例: "category": "技術" | "契約" | "一般"
ヒント ②
import json をファイル先頭に追加してください
json.loads(response.choices[0].message.content) で辞書型に変換できます

Step 2

タスク / 評価基準 / 禁止事項を含むシステムプロンプト設計

Axxx Japanの社内 FAQ 回答ボット用のシステムプロンプトを、タスク・評価基準・禁止事項の3要素を含めて設計してください。

# Step 2: タスク・評価基準・禁止事項の3点セット
system_prompt = """  # EDIT HERE!
## タスク
Axxx Japanの社内FAQ回答ボットとして、従業員の質問に答えてください。

## 評価基準（良い回答の条件）
- 200字以内に収める
- 箇条書きを使う場合は最大3点まで
- 不確かな情報は「確認が必要です」と明記する

## 禁止事項
- 社外の情報（ニュース・競合他社情報など）を引用しない
- 架空のルールや手続きを作り上げない
- 英語で回答しない（必ず日本語で）
"""

test_questions = [
    "有給休暇の申請はどこから行いますか？",
    "テレワーク中の経費精算ルールを教えてください",
]

for question in test_questions:
    response = client.chat.completions.create(
        model=MODEL,
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": question}
        ]
    )
    print(f"\n質問: {question}")
    print(f"回答: {response.choices[0].message.content}")
ヒント
## タスク → 何をするかを1〜2文で明記
## 評価基準 → 良い回答の条件を箇条書きで3〜5点
## 禁止事項 → してはいけないことを具体的に列挙（禁止事項は末尾に置くと効果的）

Step 3

Few-shot 例を含むプロンプト

問い合わせ文の感情（ポジティブ / ネガティブ / 混在）を分類するプロンプトに、Few-shot 例を3つ追加してください。

# Step 3: Few-shot プロンプト
few_shot_prompt = """  # EDIT HERE!
以下の問い合わせ文の感情を「ポジティブ」「ネガティブ」「混在」のいずれかに分類してください。

## 例
入力: 「サポートの対応がとても丁寧でした。ありがとうございます」
出力: ポジティブ

入力: 「何度問い合わせても返信がなく、非常に困っています」
出力: ネガティブ

入力: 「機能自体は良いのですが、価格が高すぎると感じます」
出力: 混在

## 分類対象
入力: {inquiry_text}
出力:
"""

test_cases = [
    "迅速に対応いただきましたが、解決策が的外れでした",
    "今まで使ったツールの中で最高です！",
    "毎回バグが出て使い物になりません",
]

for case in test_cases:
    prompt = few_shot_prompt.format(inquiry_text=case)
    response = client.chat.completions.create(
        model=MODEL,
        messages=[{"role": "user", "content": prompt}],
        max_tokens=20
    )
    label = response.choices[0].message.content.strip()
    print(f"入力: {case[:30]}... → {label}")
ヒント
各 shot は「入力: ～\n出力: ～」の形式で記述します
edge case（「混在」など判断が難しいケース）を含めると精度が上がります
最後の「入力: {inquiry_text}\n出力:」で LLM に続きを書かせます

Step 4

Structured Outputs（response_format=json_object）

Pydantic モデルを定義し、Structured Outputs を使って型安全な JSON 出力を取得してください。会議メモから「決定事項・アクションアイテム・次回日程」を抽出します。

# Step 4: Structured Outputs with Pydantic
from pydantic import BaseModel

class MeetingMinutes(BaseModel):  # EDIT HERE!
    decisions: list[str]       # 決定事項リスト
    action_items: list[str]    # アクションアイテムリスト
    next_meeting: str          # 次回会議日程

meeting_memo = """
2026年4月17日 定例会議メモ
- 新システムの本番移行日を5月1日に決定した
- 田中さんがテスト環境の構築を4月25日までに完了する
- 山田さんがユーザーマニュアルを4月28日までに作成する
- 次回会議は4月24日（金）14:00を予定
"""

response = client.beta.chat.completions.parse(  # EDIT HERE!
    model=MODEL,
    messages=[
        {"role": "system", "content": "会議メモから情報を抽出してください"},
        {"role": "user", "content": meeting_memo}
    ],
    response_format=MeetingMinutes
)

result = response.choices[0].message.parsed
print("決定事項:", result.decisions)
print("アクション:", result.action_items)
print("次回:", result.next_meeting)
ヒント ①
class MeetingMinutes(BaseModel): でモデルを定義します
フィールドは型アノテーションで定義: decisions: list[str]
ヒント ②
client.beta.chat.completions.parse() を使います（create ではなく parse）
response_format=MeetingMinutes（クラスをそのまま渡す）
結果は response.choices[0].message.parsed で取得
