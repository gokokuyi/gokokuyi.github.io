Step 4 確認テスト — カスタマイズと高度な設定

所要時間: 約10分  |  問題数: 10問

選択問題

Q1. settings.json でツールの実行を完全にブロックするための設定はどれですか？

正解: b — "deny" リストに登録したツールやコマンドは完全にブロックされ、ユーザーの確認なしに実行されることもありません。

Q2. Hooks の PostToolUse はいつ実行されますか？

正解: b — フック種別: PreToolUse（実行前）・PostToolUse（実行後）・Notification（通知時）・Stop（応答完了時）。PostToolUse はツールが完了してから発火します。

Q3. MCP（Model Context Protocol）の目的として正しいものはどれですか？

正解: b — MCP は Claude Code のツールセットを拡張するプロトコルです。Slack・GitHub・社内 API など外部サービスを Claude Code から直接利用できるようになります。

Q4. グローバル設定の settings.json が置かれる場所はどれですか？

正解: b — グローバル設定は ~/.claude/settings.json です。プロジェクト固有の設定は .claude/settings.json（プロジェクトルート内）に置きます。

Q5. CLAUDE.md が最も高い優先度で読み込まれる場所はどれですか？

正解: c — 現在の作業ディレクトリに近い（より具体的な）CLAUDE.md ほど優先されます。サブディレクトリ → プロジェクトルート → グローバルの順です。

Q6. 次の Hooks 設定が実現することはどれですか？

{"hooks": {"PostToolUse": [{"matcher": "Edit|Write", "hooks": [{"type": "command", "command": "ruff check --fix $FILE"}]}]}}
正解: b — PostToolUse（実行後）かつ Edit|Write（編集・作成後）にトリガーされ、ruff check --fix で自動修正します。PreToolUse ではないので「編集前」ではありません。

Q7. Claude Code のメモリに保存するのが適切でないものはどれですか？

正解: c — メモリは将来の会話でも使える恒久的な情報を保存する場所です。デバッグセッションの一時的な状態は揮発的であり、メモリへの保存は不適切です。

Q8. settings.json の "allow" リストに設定できることとして正しいものはどれですか？

正解: a — "allow" リストに登録したツール・コマンドはユーザーへの確認なしに自動実行されます。例えば "Bash(npm test)" を allow すると、テスト実行が自動承認されます。

Q9. カスタム MCP サーバーを作成するメリットとして最も適切なものはどれですか？

正解: b — カスタム MCP サーバーを作成することで、社内の独自 API・データベース・ツールを Claude Code のツールとして利用できるようになります。

Q10. スキル（スラッシュコマンド）の主な用途はどれですか？

正解: b — スキル（/.claude/commands/ に置く Markdown ファイル）は、繰り返し使うプロンプトや処理手順をスラッシュコマンドとして定型化する仕組みです。
