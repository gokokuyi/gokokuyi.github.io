1 CLAUDE.md の詳細活用

CLAUDE.md はプロジェクト固有の「指示書」です。Claude Code が作業を始める際に自動的に読み込まれ、動作をカスタマイズします。

読み込み順（優先度）

場所優先度
サブディレクトリ内の CLAUDE.md（例: src/CLAUDE.md）高（最優先）
プロジェクトルートの CLAUDE.md中
~/.claude/CLAUDE.md（グローバル設定）低

実践的な CLAUDE.md の例

# 社内決済システム API

## 概要
Python (FastAPI) + PostgreSQL の決済処理 API

## よく使うコマンド
- テスト: `pytest tests/ -v`
- 起動: `uvicorn main:app --reload`
- マイグレーション: `alembic upgrade head`

## コーディング規約
- Python 3.11 以上の文法を使用
- 型ヒントは必須（mypy strict モード準拠）
- エラーハンドリングは HTTPException を使う

## アーキテクチャ
- `src/routers/`  : API エンドポイント定義
- `src/services/` : ビジネスロジック
- `src/models/`   : DB モデル（SQLAlchemy）

## 禁止事項
- 本番 DB への直接操作は絶対禁止
- .env ファイルは絶対にコミットしない
- main ブランチへの直接プッシュ禁止
- 金額計算には Decimal 型を使うこと（float 禁止）

2 settings.json — 動作設定

Claude Code の動作を細かく設定するファイルです。

設定ファイルの場所

種別場所
グローバル設定~/.claude/settings.json
プロジェクト設定プロジェクトルート/.claude/settings.json

権限設定の例

{
  "permissions": {
    "allow": [
      "Read",
      "Edit",
      "Bash(git status)",
      "Bash(git log *)",
      "Bash(pytest *)"
    ],
    "deny": [
      "Bash(git push --force*)",
      "Bash(git reset --hard*)",
      "Bash(rm -rf *)"
    ]
  },
  "env": {
    "PYTHONPATH": "src",
    "NODE_ENV": "development"
  }
}

3 Hooks — 自動化ワークフロー

Hooks は特定のイベントが発生したときに自動でシェルコマンドを実行する仕組みです。

Hooks の種類

Hookタイミング
PreToolUseツール実行前
PostToolUseツール実行後
Notification通知発生時
StopClaude Code 停止時

実用的な Hooks の例

① ファイル保存後に自動フォーマット

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "python -m black $CLAUDE_TOOL_INPUT_FILE_PATH 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}

② 操作ログの記録

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "echo \"$(date): $CLAUDE_TOOL_INPUT_COMMAND\" >> /tmp/claude_log.txt"
          }
        ]
      }
    ]
  }
}

4 MCP (Model Context Protocol)

MCP は Claude Code のツールを拡張する仕組みです。外部サービスやカスタムツールを Claude Code に追加できます。

MCP で拡張した Claude Code のイメージ

通常の Claude Code:
  [Read][Write][Edit][Glob][Grep][Bash]...

MCP で拡張後:
  [Read][Write][Edit]...[Slack送信][Jira更新][DB操作][社内API呼び出し]

MCP の設定（settings.json）

{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxxx"
      }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "POSTGRES_CONNECTION_STRING": "postgresql://user:pass@localhost/mydb"
      }
    }
  }
}

よく使われる MCP サーバー

MCP サーバー提供される機能
GitHub MCPPR 操作、Issue 管理、コードレビュー
PostgreSQL MCPDB スキーマ確認、クエリ実行
Slack MCPメッセージ送信、チャンネル管理
Filesystem MCP拡張ファイルアクセス

5 スキルとスラッシュコマンド

繰り返し使う処理をコマンドとして定型化できます。

// ~/.claude/settings.json
{
  "skills": {
    "review": {
      "description": "コードレビューを実施する",
      "prompt": "このコードをレビューして。セキュリティ・パフォーマンス・可読性の観点で"
    },
    "commit": {
      "description": "変更をコミットする",
      "prompt": "git diff を確認して、適切なコミットメッセージを作成してコミットして"
    }
  }
}
