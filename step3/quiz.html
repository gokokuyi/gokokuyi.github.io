Step 3 確認テスト — エージェント機能と自動化

所要時間: 約10分  |  問題数: 10問

選択問題

Q1. Claude Code のサブエージェントについて正しい説明はどれですか？

正解: b — サブエージェントはメイン Claude が自律的に起動する専門的な AI です。複雑な作業を分担・並列処理するために使われます。

Q2. コードベースの素早い探索に特化したサブエージェントはどれですか？

正解: c — Explore エージェントはコードベースの高速探索に特化しています。ファイル検索・コード検索・構造の把握を素早く行います。

Q3. Plan エージェントの特徴として正しいものはどれですか？

正解: b — Plan エージェントは 読み取り専用です。Edit/Write などのファイル変更ツールは持たず、実装計画の立案に専念します。

Q4. TodoWrite によるタスク管理について正しい説明はどれですか？

正解: b — TodoWrite ツールで Claude 自身がタスク一覧を作成・更新します。ユーザーが進捗をリアルタイムに確認でき、大きな作業の抜け漏れを防ぎます。

Q5. 並列処理が最も効果的でない場面はどれですか？

正解: c — Read の結果を踏まえて Edit するなど、前の結果が次の入力になる「順序依存の作業」は並列化できません。逆に独立した調査や検索は並列処理が有効です。

Q6. サブエージェントへの指示で注意すべきことは？

正解: b — サブエージェントはメイン会話の文脈を引き継ぎません。指示には「何を調査すべきか・なぜ必要か・どんな情報を返すか」を含めた完結した内容が必要です。

Q7. プロジェクト全体のセキュリティレビューをしたい場合、最も適切なアプローチはどれですか？

正解: b — Explore エージェントがコードベースを横断的に調査してメインのコンテキストを節約しつつ、結果の要点をメインに返してレポートを作成するのが最適です。

Q8. タスクの状態「→」（矢印）は何を意味しますか？

正解: c — TodoWrite のタスク状態: ✓ = 完了、→ = 実行中、空 = 未着手。矢印は現在進行中であることを示します。

Q9. エージェントを使うとコンテキストを節約できる理由はどれですか？

正解: b — サブエージェントは独立したコンテキストで膨大な調査を行い、その結果の要点のみをメインに返します。メインのコンテキストウィンドウを消費せずに大規模な調査が可能です。

Q10. バックグラウンド実行が適しているタスクはどれですか？

正解: b — バックグラウンド実行は完了まで待つ必要がない長時間タスクに適しています。完了時に通知を受け取れるため、その間に他の作業を進められます。
