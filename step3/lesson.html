Step 3: エージェント機能と自動化

対象: 中級  |  所要時間: 約60分

ラーニング目標

  • Agent ツールとサブエージェントの概念を理解する
  • 利用可能なサブエージェントの種類と使い分けを習得する
  • TodoWrite でタスク管理する方法を学ぶ
  • 並列処理とバックグラウンド実行の活用法を理解する

1 エージェント機能とは

Claude Code では、複雑なタスクを処理するために「サブエージェント」を起動できます。メインの Claude が、専門的なサブエージェントに作業を委任する仕組みです。

ユーザーの依頼
    ↓
メイン Claude（調整役）
    ↓
┌──────────────────────────────────┐
│  サブエージェントA（調査担当）    │
│  サブエージェントB（実装担当）    │
│  サブエージェントC（テスト担当）  │
└──────────────────────────────────┘
    ↓
結果をまとめてユーザーに報告

エージェントを使うメリット

状況メリット
大規模なコードベースの探索メインのコンテキストを節約できる
独立した複数タスク並列処理で高速化
専門的な作業専用エージェントの知識を活用
長時間タスクバックグラウンド実行で並行作業

2 サブエージェントの種類

汎用

general-purpose エージェント

複雑なマルチステップタスク、コードベースの調査に使う。全ツールにアクセス可能。特定のファイルや関数を探すのが難しい場合に活用。

探索専用

Explore エージェント

コードベースの素早い探索に特化。読み取り専用（編集ツールなし）。ファイルパターン検索・キーワード検索を効率的に実行。

設計専用

Plan エージェント

実装計画の立案・設計に特化。読み取り専用（実装はしない）。大きな機能追加・リファクタリングの前に使う。

知識参照

claude-code-guide エージェント

Claude Code 自体の使い方を調べるときに使う。Claude Code のドキュメントに特化。MCP 設定、Hooks 設定などの方法を調べるときに活用。

3 TodoWrite — タスク管理

複雑なタスクを実行する際、Claude Code は TodoWrite ツールで作業を管理します。タスクの進捗が可視化されるため、何をしているか把握しやすくなります。

[タスクリスト]
✓ ファイル構造を確認する         ← 完了
✓ 既存の認証コードを読み込む      ← 完了
→ 新しい認証関数を実装する        ← 現在実行中
○ テストを追加する               ← 待機中
○ ドキュメントを更新する          ← 待機中

タスクの状態

状態記号意味
完了✓終了したタスク
実行中→現在進行中
待機○まだ実行していない

4 並列処理の活用

独立した複数のタスクを同時に実行することで、作業を高速化できます。

You: src/auth.py と src/utils.py と tests/ を同時に調査して、
     問題点をまとめてレポートして

[3つのサブエージェントが並列起動]
  エージェントA: src/auth.py を調査
  エージェントB: src/utils.py を調査
  エージェントC: tests/ を調査
  → 結果をまとめて報告

並列 vs 順序実行の判断

並列でOK（独立したタスク）順序が必要（依存するタスク）
複数のファイルを同時に調査ファイルを読んでから編集
異なるモジュールを同時にテストテストを書いてから実行
複数 API を同時に調査設計してから実装

5 バックグラウンド実行

時間のかかるタスクをバックグラウンドで実行し、その間に他の作業を続けられます。

You: 全ファイルのコードレビューをして（バックグラウンドで）

[バックグラウンドでエージェント起動]

You: （別の作業を続ける）

[レビュー完了後、通知が来る]
Claude: バックグラウンドのレビューが完了しました。
        問題点が7箇所見つかりました...

6 効果的なエージェント活用パターン

パターン1: 調査 → 計画 → 実装

Step 1: 「このバグの原因を調査して（広く調べて）」
Step 2: 「調査結果をもとに修正計画を立てて」
Step 3: 「計画に従って修正を実装して」

パターン2: コード品質チェック

「プロジェクト全体を調査して：
 - セキュリティ上の問題
 - パフォーマンスのボトルネック
 - テストカバレッジが低い箇所
 をまとめてレポートして」

7 エージェントへの効果的な指示

NGOK
「バグを探して」「src/ 以下を調査して、型ヒントがない関数をリストアップして」
「さっき話した件を調査して」「src/auth.py の login 関数において、None チェックがないという問題を調査して」

まとめ

機能用途効果
