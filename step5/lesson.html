Step 5: チーム活用とベストプラクティス

対象: 全員  |  所要時間: 約60分

ラーニング目標

  • チームでの Claude Code 活用ルールを設計できる
  • セキュリティリスクを理解し、適切に対処できる
  • 効果的なプロンプト設計の原則を習得する
  • CI/CD への統合方法を理解する

1 チームでの運用設計

段階的な導入アプローチ

Phase 1: 個人ラーニング（1〜2週間）
  • 各自が個人の作業で試す
  • 便利な使い方を発見する
  • 不安な点を洗い出す
Phase 2: ルール整備（1週間）
  • 利用ガイドラインを作成
  • CLAUDE.md を整備
  • 禁止事項を明確にする
Phase 3: チーム展開（継続）
  • 全員に展開
  • 定期的に活用事例を共有
  • ルールを継続的に改善

チームで合意すべき主なポイント

カテゴリ決めること
使って良い操作自動承認できる操作の範囲
確認必須の操作必ずレビューが必要な操作
禁止操作絶対にやらない操作
コードレビューAI 生成コードのレビュー方針
機密情報Claude Code に渡してよい情報の範囲
コスト管理API 使用量の監視・上限

2 セキュリティの考慮事項

渡してはいけない情報

NG（渡してはいけない）OK（渡して良い）
本番環境の API キー・パスワード開発環境のコード・設定
個人情報（氏名・メール・住所等）匿名化されたサンプルデータ
社外秘の設計書・仕様書公開情報・ドキュメント
顧客データ・財務情報テストコード

git 操作のリスク管理

高リスク（必ず確認）低リスク（自動承認 OK）
git push --forcegit status
git reset --hardgit log
git branch -Dgit diff
main/master への直接コミットfeature ブランチへの commit

3 効果的なプロンプト設計

良いプロンプトの5原則

① 具体的に指示する

NGOK
「このコードを改善して」「src/auth.py の login() 関数に、パスワード最小長チェック（8文字以上）を追加して」

② コンテキストを与える

NGOK
「エラーを修正して」「以下のエラーが pytest 実行時に発生しています：
AttributeError: 'NoneType' object has no attribute 'email'
テストファイル: tests/test_user.py の test_create_user 関数」

③ 出力形式を指定する

「セキュリティ上の問題点を以下の形式でまとめて：
  - 重大度: 高/中/低
  - 場所: ファイル名と行番号
  - 問題: 問題の説明
  - 対策: 推奨される修正方法」

④ 制約を伝える

「以下の制約でリファクタリングして：
  - 既存のテストを壊さない
  - 外部 API の仕様は変更しない（後方互換性を維持）
  - Python 3.10 以前でも動作すること」

⑤ 段階的に進める

Step 1: 「まず現在の認証システムの問題点を分析して」
Step 2: 「分析結果をもとに改善計画を立てて」
Step 3: 「計画の優先度が高い部分から実装して」

4 CI/CD への統合

GitHub Actions での活用例

# .github/workflows/claude-review.yml
name: Claude Code Review
on:
  pull_request:
    types: [opened, synchronize]
jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Claude Code Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          npm install -g @anthropic-ai/claude-code
          claude --no-interactive \
            "この PR のコード変更をレビューして、問題点を指摘して"

自動化に向く/向かないタスク

自動化に向くタスク注意が必要なタスク
読み取りのみの操作本番環境への変更
繰り返し同じ作業をするもの外部サービスへの書き込み
人間のチェックが後に入るもの機密情報を扱う操作

5 トラブルシューティング

問題対処
意図しない操作が実行されたgit restore . でリセット → CLAUDE.md に禁止事項を追加
コンテキストが長くなりすぎる/clear でリセット → タスクを小さく分割
繰り返し同じ間違いをするCLAUDE.md に明示 → メモリに記録
生成されたコードが動かないエラーメッセージをそのまま渡す → 環境情報を伝える
API コストが高い/cost で確認 → /clear でリセット → 使用ポリシーを見直す

6 AI と協働するマインドセット

✓ AI の提案を必ず確認してから適用する
✓ 理解できないコードは「説明して」と聞く
✓ エラーが出たら丸ごとエラーメッセージを渡す
✓ 良い結果が出たら「その方針で続けて」と伝える
✓ git で安全網を張っておく
